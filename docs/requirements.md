# 高校实验室预约管理系统 — 需求规格文档

> 版本：1.0 | 日期：2026-06-29 | 作者：Claude

---

## 一、项目概述

基于 Spring Boot + Vue 3 的高校实验室预约管理系统，面向高校实验室日常管理场景，为**学生、教师、实验室管理员**三类用户提供从实验室浏览、预约审批、设备借还到统计分析的全流程数字化管理。

---

## 二、目标用户

| 角色             | 描述                                                                                                     |
| ---------------- | -------------------------------------------------------------------------------------------------------- |
| **学生**         | 在校本科生/研究生，通过系统浏览实验室、提交预约申请、借用/归还设备、查看课表和通知、对实验室进行评价     |
| **教师**         | 授课教师，审批学生预约、安排实验课程、发布通知、查看使用记录和统计报表                                   |
| **实验室管理员** | 实验室日常管理责任人，管理实验室基础信息、管理设备资产、审批预约和借用、设置开放时间、查看全系统统计报表 |

## 三、功能需求清单

### 3.1 用户认证与权限管理

| 编号 | 功能             | 优先级 | 说明                                                                      |
| ---- | ---------------- | ------ | ------------------------------------------------------------------------- |
| F-01 | 用户注册         | P0     | 默认注册为 STUDENT 角色，用户名唯一（学号/工号）                          |
| F-02 | 用户登录         | P0     | JWT 认证，返回 token 和用户基本信息                                       |
| F-03 | 获取当前用户信息 | P0     | 用于前端路由守卫和页面权限判断                                            |
| F-04 | 修改密码         | P1     | 需验证原密码                                                              |
| F-05 | 角色权限控制     | P0     | 基于 RBAC 的三级角色（STUDENT/TEACHER/ADMIN），不同角色看到不同菜单和功能 |
| F-06 | 用户管理         | P1     | ADMIN 可查看用户列表、禁用/启用账号                                       |

### 3.2 实验室信息管理

| 编号 | 功能           | 优先级 | 说明                                                                     |
| ---- | -------------- | ------ | ------------------------------------------------------------------------ |
| F-07 | 实验室列表查询 | P0     | 支持分页、按名称/状态筛选，所有角色可访问                                |
| F-08 | 实验室详情查看 | P0     | 展示完整信息、开放时间、设备清单、评价列表                               |
| F-09 | 实验室 CRUD    | P0     | ADMIN 专属：创建、编辑、删除实验室信息                                   |
| F-10 | 开放时间设置   | P0     | ADMIN 为每个实验室设置每周开放时间段（支持每天多时段）                   |
| F-11 | 实验室状态管理 | P1     | 标记实验室状态：AVAILABLE（可用）/ MAINTENANCE（维护中）/ CLOSED（关闭） |

### 3.3 预约管理（核心）

| 编号 | 功能           | 优先级 | 说明                                                           |
| ---- | -------------- | ------ | -------------------------------------------------------------- |
| F-12 | 提交预约申请   | P0     | 学生/教师选择实验室、日期、时间段、填写用途和人数              |
| F-13 | 可用时间段查询 | P0     | 给定实验室和日期，返回可预约时间段（排除已审批预约和课程占用） |
| F-14 | 预约审批       | P0     | 教师/管理员审批：通过或驳回（需填写驳回原因）                  |
| F-15 | 我的预约列表   | P0     | 学生/教师查看自己的预约记录和状态                              |
| F-16 | 待审批列表     | P0     | 教师/管理员查看待审批的预约申请                                |
| F-17 | 取消预约       | P1     | 申请人可取消 PENDING 或 APPROVED 状态的预约                    |
| F-18 | 修改预约       | P1     | 仅 PENDING 状态可修改                                          |
| F-19 | 确认完成       | P1     | 预约使用完毕后标记为 COMPLETED，自动生成使用记录               |
| F-20 | 时间冲突检测   | P0     | 审批时自动检测：是否与已审批预约 + 课程安排有时间重叠          |

**审批状态机**：

```
PENDING → APPROVED → COMPLETED
PENDING → REJECTED（需填写驳回原因）
PENDING → CANCELLED（申请人自行取消）
APPROVED → CANCELLED（申请人自行取消）
```

## 四、非功能需求

### 4.1 安全性

| 编号  | 需求         | 说明                                        |
| ----- | ------------ | ------------------------------------------- |
| NF-01 | 认证机制     | 基于 JWT 的无状态认证，token 有效期 24 小时 |
| NF-02 | 权限控制     | 前端路由守卫 + 后端方法级权限注解，双重校验 |
| NF-03 | SQL 注入防护 | 全部使用 MyBatis-Plus 参数化查询            |
| NF-04 | XSS 防护     | 前端输入过滤 + 后端响应转义                 |
| NF-05 | 密码安全     | BCrypt 加密存储，不记录明文                 |
| NF-06 | 接口鉴权     | 除登录注册外所有接口需携带有效 JWT          |

### 4.2 性能与可用性

| 编号  | 需求     | 说明                                           |
| ----- | -------- | ---------------------------------------------- |
| NF-07 | 响应时间 | 常规查询接口 < 500ms                           |
| NF-08 | 缓存策略 | Redis 缓存高频读取数据（实验室列表、用户信息） |
| NF-09 | 并发控制 | 预约审批环节使用 Redis 分布式锁防止重复审批    |

### 4.3 可维护性

| 编号  | 需求         | 说明                                                            |
| ----- | ------------ | --------------------------------------------------------------- |
| NF-10 | 统一响应格式 | 所有 API 返回 `{ code, message, data }`                         |
| NF-11 | 全局异常处理 | `@RestControllerAdvice` 统一处理业务异常和参数校验异常          |
| NF-12 | 参数校验     | 前端 Element Plus 表单校验 + 后端 Bean Validation 双重校验      |
| NF-13 | 代码规范     | TypeScript strict 模式，ESLint + Prettier，Conventional Commits |
| NF-14 | 日志记录     | 关键操作（审批、删除）记录日志                                  |

### 4.4 兼容性

| 编号  | 需求       | 说明                                        |
| ----- | ---------- | ------------------------------------------- |
| NF-15 | 浏览器兼容 | Chrome/Firefox/Edge 最新两个主版本          |
| NF-16 | 部署方式   | Docker Compose 一键部署，支持 Linux/Windows |

## 六、MVP 范围界定

### P0 — 必须实现（核心闭环，Day 1-5）

- 用户注册/登录 + JWT 认证 + RBAC 三级权限
- 实验室列表/详情 + 管理员 CRUD + 开放时间设置
- **预约申请 + 审批流程**（PENDING → APPROVED/REJECTED，含时间冲突检测）
- 可用时间段查询
- 设备列表/详情 + 管理员 CRUD
- 设备借用申请 + 审批 + 归还确认
- 课程安排 + 学生课表查看
- 通知公告发布与查看
- 各角色仪表盘

### P1 — 应该实现（增强体验，Day 6-7）

- 我的评价 / 提交评价
- 使用记录查询
- 综合统计报表 + ECharts 图表
- Redis 缓存
- 修改密码
- 用户管理（管理员）
- 预约取消/修改
- 实验室状态管理

### P2 — 可以后做（锦上添花，Day 8-10 视进度）

- 学生使用排行
- 使用记录 CSV 导出
- 设备利用率统计
- 我的评价历史

## 七、附录

### 术语表

| 术语                       | 说明                                                   |
| -------------------------- | ------------------------------------------------------ |
| 预约 (Booking)             | 学生对实验室在特定日期和时间段的使用申请               |
| 审批 (Approval)            | 教师/管理员对预约申请的审核操作                        |
| 开放时间 (Lab Hours)       | 实验室每周固定的可预约时间段                           |
| 课程安排 (Course Schedule) | 教师安排的周期性实验课程，与单次预约属于不同的排程维度 |
| 时间冲突 (Time Conflict)   | 同一实验室、同一日期、时间段存在重叠的已审批预约或课程 |
| 使用记录 (Usage Record)    | 预约完成后的归档记录，用于追溯和统计                   |

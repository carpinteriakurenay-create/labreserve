## 二、实现计划

### 执行顺序

```
M3-3 (设备管理增强) → M3-4 (借用申请增强) → M3-5 (时间线重写) → M3-6 (借用管理增强)
```

M3-3 和 M3-4 相互独立可并行，M3-5 和 M3-6 相互独立可并行。按编号顺序推进，每完成一个即可验证。

### 关键设计决策

| 决策                                                | 理由                                                                                                                     |
| --------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| 序列号异步唯一性校验由后端 API 层统一处理           | 前端行内校验仅做长度检查，后端返回 `SERIAL_NUMBER_EXISTS` → 409 时由 Axios 拦截器统一 toast 提示，与现有错误处理模式一致 |
| 设备 BORROWED 状态不显示报修按钮                    | 借出的设备需先归还才能报修，避免状态冲突                                                                                 |
| 归还日期弹窗默认填今天                              | 管理员通常当天确认归还，减少操作步骤                                                                                     |
| 我的借用页用 `el-timeline` 替代 `el-table`          | 借用记录是时间序列数据，时间线更直观地展示生命周期；规范明确要求"时间线展示"                                             |
| 时间线节点颜色用内联 `:color` 绑定                  | `el-timeline-item` 的 `color` prop 接受 CSS 颜色字符串，比 CSS 类名更直接                                                |
| 页面状态全部用本地 `ref`/`reactive` 管理            | 这四个页面不需要跨组件共享状态，无需 Pinia store；与项目现有模式一致                                                     |
| 所有操作保留 `ElMessageBox.confirm` 确认            | M3-3 报修、M3-6 审批通过等操作不可逆，需用户二次确认                                                                     |
| 归还日期通过 `Date.toISOString().slice(0, 10)` 获取 | 避免 `split("T")[0]` 的 TypeScript `undefined` 类型推断问题                                                              |

### M3-3 (AdminEquipmentView) 验证

1. **登记设备** — 点击"登记设备"，填写表单，保存后在列表中可见
2. **序列号校验** — 创建时输入 1 个字符序列号，失去焦点后出现"序列号至少 2 个字符"行内错误
3. **重复序列号** — 创建时使用已存在的序列号，提交后 toast 提示"SERIAL_NUMBER_EXISTS"
4. **编辑设备** — 点击编辑，修改名称/型号/描述，保存后列表刷新
5. **报修操作** — AVAILABLE 设备点击"报修" → 确认弹窗 → 状态变为"维修中"（红色标签）
6. **恢复可用** — MAINTENANCE 设备点击"恢复可用" → 自动恢复 → 状态变为"可用"（绿色标签）
7. **BORROWED 设备** — 操作列仅显示"编辑"和"删除"，不显示"报修"
8. **删除确认** — 点击删除 → 确认弹窗 → 设备从列表消失

### M3-4 (BorrowRequestView) 验证

1. **设备选择器** — 下拉列表仅显示 AVAILABLE 状态的设备
2. **设备信息卡片** — 选中设备后，表单下方出现设备信息卡片（名称、型号、序列号、实验室、状态标签）
3. **借用日期约束** — 点击日期选择器，过去日期灰色不可选
4. **归还日期约束** — 选择借用日期后，归还日期选择器中早于/等于借用日期的日期灰色不可选
5. **日期联动** — 选择借用日期 → 重新选择更晚的借用日期 → 原归还日期自动清除
6. **借用天数提示** — 两个日期都选择后，"预计借用 N 天"灰色文字显示
7. **提交申请** — 填写完整 → 提交 → toast "借用申请已提交" → 表单重置

### M3-5 (MyBorrowsView) 验证

1. **时间线展示** — 页面以时间线形式展示借用记录（非表格），节点颜色与状态对应
2. **卡片内容** — 每张卡片包含设备名称 + 状态标签 + 借用日期 + 预计归还 + 用途
3. **状态颜色** — PENDING=蓝色、APPROVED/BORROWING=绿色、REJECTED=红色、RETURNED=灰色
4. **拒绝原因** — REJECTED 记录的卡片显示红色拒绝原因文字
5. **实际归还** — RETURNED 记录显示实际归还日期
6. **状态筛选** — 下拉选择状态 → 查询 → 仅显示对应状态记录
7. **重置筛选** — 点击重置 → 状态筛选清空 → 显示全部记录
8. **空状态** — 无借用记录时显示 `el-empty` "暂无借用记录"
9. **分页** — 记录超过 20 条时显示分页器

### M3-6 (AdminBorrowsView) 验证

1. **三个 Tab** — 待审批 / 借用中 / 全部，切换时数据正确筛选
2. **审批通过** — 待审批 Tab 点击"通过" → 确认弹窗 → 记录状态变为 BORROWING
3. **审批拒绝** — 点击"拒绝" → 弹窗填写拒绝原因 → 确认 → 记录状态变为 REJECTED
4. **拒绝原因 tooltip** — 拒绝原因超过 20 字时截断显示，鼠标悬停显示完整内容
5. **归还弹窗** — 借用中记录点击"确认归还" → 弹窗选择归还日期（默认今天）→ 确认 → 记录状态变为 RETURNED
6. **全部 Tab 操作** — 切换到"全部"Tab，PENDING 记录仍显示"通过/拒绝"按钮，BORROWING 记录仍显示"确认归还"
7. **审批确认** — 通过操作有 `ElMessageBox.confirm` 二次确认
8. **分页** — 记录多时正确分页

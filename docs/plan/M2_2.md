## 二、实现计划

### 执行顺序

```
labs.ts API 层 ──> router 路由 ──> AppLayout 菜单
                                       │
         ┌─────────────────────────────┤
         │                             │
  LabsView.vue (M2-5)          AdminLabsView.vue (M2-6)
         │
  LabDetailView.vue (M2-5)
```

Step 1-3（API + Router + Menu）必须先完成，Step 4-6 可并行。

### 关键设计决策

| 决策                                   | 理由                                                                 |
| -------------------------------------- | -------------------------------------------------------------------- |
| **列表页用卡片网格**                   | 用户浏览体验更直观，区别于管理页的表格                               |
| **管理页用 Tabs**                      | CRUD 和开放时间放同一页面减少导航跳转，符合常见 admin 模式           |
| **分页参数用 pageNum/pageSize**        | 后端 MyBatis-Plus IPage 使用此格式，共享类型的 cursor/limit 暂不适用 |
| **toggleStatus 仅 AVAILABLE ↔ CLOSED** | 后端 toggleStatus 逻辑如此，MAINTENANCE 通过 updateLab 显式设置      |
| **LabDetailView 设备区占位**           | 设备 API 属 M3 范围，当前仅显示设备数量                              |
| **MenuItem.icon 类型改为 Component**   | 支持多种 Element Plus 图标                                           |
| **DAY_OF_WEEK_LABELS 局部定义**        | 1-7 → "周一"~"周日"，纯展示逻辑，暂不提取到 shared                   |

### TimeSlotPicker 组件（M2-11）

- **Props**: `labId` (number | string | null)、`date` (string | null)
- **Emit**: `update:selectedSlot` → `{ startTime, endTime } | null`
- **逻辑**: `watch([labId, date])` → 两 prop 有效时调 `getAvailableSlots`；prop 变化时清除选中并重新 fetch
- **渲染**: CSS Grid 4 列，每格 `HH:mm - HH:mm`
  - 可用 → `#f0f9eb` 背景 / `#67c23a` 文字 / 指针
  - 不可用 → `#f5f7fa` 背景 / `#c0c4cc` 文字 / `not-allowed`
  - 选中 → `#ecf5ff` 背景 / `#409eff` 边框
- **空状态**: `el-empty` "请先选择实验室和日期"（无 prop）或 "该日期暂无可用时段"（API 返回空）

### BookingCreateView 页面（M2-10）

全页表单（非弹窗），5 个字段：

| 字段     | 组件                                                                   | 校验                   |
| -------- | ---------------------------------------------------------------------- | ---------------------- |
| 实验室   | `el-select` 加载全部实验室                                             | 必填                   |
| 预约日期 | `el-date-picker`，`disabledDate` 禁用过去，`value-format="YYYY-MM-DD"` | 必填                   |
| 时间段   | `TimeSlotPicker`（v-model:selectedSlot）                               | 必填，自定义 validator |
| 预约用途 | `el-input textarea`，max 500，show-word-limit                          | 必填                   |
| 使用人数 | `el-input-number`，min=1，max=999，默认 1                              | —                      |

交互要点：

- 从 query param `?labId=X` 预选实验室
- `submitting` ref 防重复提交
- 成功 → `ElMessage.success` → `router.push("/bookings/mine")`
- TIME_CONFLICT（409）→ interceptor 显示错误 → 递增 `slotRefreshKey` 强制刷新 TimeSlotPicker

### 路由 + 导航

- 路由: `/bookings/create` → BookingCreateView，所有已认证用户可访问
- LabDetailView 底部增加"预约此实验室"按钮，跳转 `/bookings/create?labId={id}`

### 关键设计决策

| 决策                               | 理由                                                            |
| ---------------------------------- | --------------------------------------------------------------- |
| **全页面而非弹窗**                 | 与现有代码库表单页面模式一致；预约表单字段较多，弹窗空间局促    |
| **TimeSlotPicker 用 key 强制刷新** | 409 冲突后需重新获取 slots；key 变化触发完整重 mount + 重 fetch |
| **Date picker 允许今天**           | 用户可能在当天预约稍晚的时段                                    |
| **lab 下拉不过滤状态**             | CLOSED/MAINTENANCE 提交时后端返回错误，interceptor 提示用户     |
| **API 层初始仅 2 函数**            | getAvailableSlots + createBooking；M2-12/M2-13 再追加其余       |

### M2-14：Axios 拦截器增强

`client.ts` 三项改动：

1. **静默模式** — `declare module "axios"` 扩展 `AxiosRequestConfig`，新增 `_silent?: boolean`，为 `true` 时拦截器不弹 `ElMessage.error`
2. **请求计数** — `activeRequests` 计数器，request interceptor `++`，response/error interceptor `--`，供外部组件读取 loading 状态
3. **401 处理** — 401 响应清除 localStorage 并跳转 `/login`，`_silent` 标记可抑制 Toast

### 关键设计决策

| 决策                                     | 理由                                                                            |
| ---------------------------------------- | ------------------------------------------------------------------------------- |
| **API 分页用 pageNum/pageSize**          | 后端 BookingController 使用 MyBatis-Plus IPage，与现有 BookingQueryParams 一致  |
| **审批页用双 Tab**                       | 待审批列表和全部预约放同一页面减少菜单项，教师/管理员在一个页面完成所有审批操作 |
| **驳回用 dialog 而非 MessageBox.prompt** | 驳回原因可能较长，dialog + textarea 体验更友好                                  |
| **`_silent` 一次性标记**                 | 某些场景（如静默轮询）不需要弹错误，配置标记即可复用现有错误处理                |
| **`activeRequests` 导出**                | 供 AppLayout 或其他组件读取，实现全局 loading bar                               |
| **状态标签颜色语义一致**                 | 橙=待处理、绿=成功、红=拒绝、灰=取消，与 Element Plus 约定对齐                  |

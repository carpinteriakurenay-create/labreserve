### M2-5：实验室列表/详情页

所有已认证用户可访问。列表页用卡片网格展示实验室（名称搜索 + 状态筛选 + 分页），点击卡片进入详情页（基本信息 + 每周开放时间表）。

### M2-6：实验室管理页

ADMIN 专属。双 Tab 页面：Tab1 为 CRUD 表格（创建/编辑弹窗、删除确认、状态切换），Tab2 为开放时间批量编辑器（选择实验室 → 7 行时间设置）。

### 路由

| 路径          | 页面          | 角色           |
| ------------- | ------------- | -------------- |
| `/labs`       | LabsView      | 所有已认证用户 |
| `/labs/:id`   | LabDetailView | 所有已认证用户 |
| `/admin/labs` | AdminLabsView | ADMIN          |

### 菜单

菜单顺序：系统首页 → 实验室列表 → 用户管理(ADMIN) → 实验室管理(ADMIN)

## 二、实现计划

### 执行顺序

```
labs.ts API 层 ──> router 路由 ──> AppLayout 菜单
                                       │
         ┌─────────────────────────────┤
         │                             │
  LabsView.vue (M2-5)          AdminLabsView.vue (M2-6)
         │
  LabDetailView.vue (M2-5)
```

Step 1-3（API + Router + Menu）必须先完成，Step 4-6 可并行。

### 关键设计决策

| 决策                                   | 理由                                                                 |
| -------------------------------------- | -------------------------------------------------------------------- |
| **列表页用卡片网格**                   | 用户浏览体验更直观，区别于管理页的表格                               |
| **管理页用 Tabs**                      | CRUD 和开放时间放同一页面减少导航跳转，符合常见 admin 模式           |
| **分页参数用 pageNum/pageSize**        | 后端 MyBatis-Plus IPage 使用此格式，共享类型的 cursor/limit 暂不适用 |
| **toggleStatus 仅 AVAILABLE ↔ CLOSED** | 后端 toggleStatus 逻辑如此，MAINTENANCE 通过 updateLab 显式设置      |
| **LabDetailView 设备区占位**           | 设备 API 属 M3 范围，当前仅显示设备数量                              |
| **MenuItem.icon 类型改为 Component**   | 支持多种 Element Plus 图标                                           |
| **DAY_OF_WEEK_LABELS 局部定义**        | 1-7 → "周一"~"周日"，纯展示逻辑，暂不提取到 shared                   |

## 三、验证方案

> **前置条件：** 后端已启动（`docker compose up -d` + `mvn spring-boot:run`），前端已启动（`pnpm dev:web`），有 seed 数据（2 个实验室 + ADMIN 用户）。

### M2-5 验证（所有用户）

1. 访问 `http://localhost:5173/labs`
2. 看到 2 个实验室卡片（来自 seed data），显示名称、位置、容量、状态标签、设备数
3. 在搜索框输入"计算机" → 过滤结果
4. 状态筛选选择"可用" → 仅显示 AVAILABLE 状态的实验室
5. 点击重置 → 恢复全部
6. 点击卡片标题 → 跳转到 `/labs/1`
7. 详情页显示：基本信息（名称、状态、位置、容量、管理员、描述）+ 开放时间表（周一~周日）
8. 点击"返回列表" → 回到 `/labs`

### M2-6 验证（ADMIN 登录）

1. 以 admin 账号登录，菜单中出现"实验室管理"
2. 访问 `/admin/labs` → 表格展示所有实验室
3. 点击"创建实验室" → 弹窗表单，填写名称+容量 → 保存 → 列表刷新
4. 点击"编辑" → 弹窗预填数据 → 修改 → 保存 → 列表刷新
5. 点击"删除" → 确认弹窗 → 删除成功
6. 点击"关闭"/"开放" → 状态标签切换
7. 切换到"开放时间管理" Tab → 选择实验室 → 显示 7 天编辑器
8. 填写部分天的开始/结束时间 → 点击保存 → 提示成功
9. 返回实验室列表页 → 点击该实验室详情 → 开放时间正确显示

### 异常场景

1. 未登录访问 `/labs` → 重定向到 `/login`
2. 非 ADMIN 用户访问 `/admin/labs` → 重定向到首页
3. 访问不存在的 `/labs/99999` → 显示"实验室不存在"
4. 详情页无开放时间 → 显示"暂无开放时间"占位
5. 删除操作取消 → 不执行删除

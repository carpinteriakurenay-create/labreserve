| 决策                                                    | 理由                                                         |
| ------------------------------------------------------- | ------------------------------------------------------------ |

| 校验 booking.status == COMPLETED 后才允许评价           | 符合 F-35 需求：只能评价已完成的预约                         |
| 校验 booking.userId == currentUserId                    | 禁止评价他人预约                                             |
| DELETE 仅限本人                                         | 与权限矩阵一致（ADMIN 可通过 DB 操作）                       |
| `populateJoinedFields` 批量加载 labName + userName      | 与 BookingService 模式一致，避免 N+1 查询                    |
| 前端 el-rate 只读展示                                   | 评分不可修改，创建后即固定                                   |

| -------------------------------------------------- | ------------------------------------------------------------- |
| Java Stream 聚合而非 SQL GROUP BY                  | 项目无 XML Mapper，保持与现有风格一致；数据量小，内存聚合即可 |
| STUDENT 角色只看自己数据                           | 通过 `userId` 过滤 booking/borrow 查询条件                    |
| `labUsageRate` = 今日有预约的实验室数 / 总实验室数 | 简单直观，无需复杂的时间段计算                                |

| 决策                                  | 理由                                                        |
| ------------------------------------- | ----------------------------------------------------------- |
| 不建 `usage_records` 物理表           | API 文档定义为查询视图，直接 JOIN 三表即可                  |
| CSV 输出 UTF-8 BOM (`﻿`)              | Windows Excel 需要用 BOM 识别 UTF-8，否则中文乱码           |
| CSV 字段双引号包裹 + 内部 `"` 转 `""` | 防止 purpose 字段中的逗号和换行破坏 CSV 结构                |
| 导出不走分页（`selectList` 全量）     | 高校实验室数据量在可控范围内，后续可改为流式导出            |
| `responseType: "blob"` 前端下载       | 二进制下载标准方式，`URL.createObjectURL` + 动态 `<a>` 触发 |
| 类级别 `@PreAuthorize` 控制全部端点   | 与 UserController 模式一致                                  |

### 分层实现步骤

**后端（每个模块）：**

```
DTO → Service → Controller
```

1. DTO 层：Request（含校验注解）+ VO（含 Builder）
2. Service 层：构造函数注入 Mapper → @Transactional 写操作 → LambdaQueryWrapper 查询 → BusinessException 错误 → toVO 转换 → populateJoinedFields 填充
3. Controller 层：构造函数注入 Service → @PreAuthorize 权限 → ApiResponse 包装 → getCurrentUserId 辅助方法

**前端（每个模块）：**

```
API 文件 → 组件 → View 页面 → 路由注册 → 菜单项
```

## 三、验证方案

### M4-1 验证

> **前置条件：** 先完成一个完整的预约流程（创建 → 审批 → 完成），至少有 1 条 COMPLETED 状态的预约。

```bash
# 获取 token（用户需有至少一个已完成预约）
TOKEN=$(curl -s -X POST localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"test001","password":"pass123"}' \
  | sed 's/.*"token":"\([^"]*\)".*/\1/')

# 1. 提交评价 → 201
curl -s -X POST localhost:8080/api/reviews \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"bookingId":1,"rating":4,"comment":"设备运行良好"}'
# → 201 {"code":"SUCCESS","message":"评价提交成功","data":{"id":1,"labName":"...","rating":4,...}}

# 2. 重复评价 → 409
curl -s -X POST localhost:8080/api/reviews \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"bookingId":1,"rating":5,"comment":"重复"}'
# → 409 {"code":"ALREADY_REVIEWED","message":"该预约已评价"}

# 3. 评价未完成的预约 → 400
curl -s -X POST localhost:8080/api/reviews \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"bookingId":<pendingId>,"rating":3}'
# → 400 {"code":"BOOKING_NOT_COMPLETED","message":"只能评价已完成的预约"}

# 4. 评分超出范围 → 400
curl -s -X POST localhost:8080/api/reviews \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"bookingId":2,"rating":6}'
# → 400 {"code":"VALIDATION_ERROR","message":"请求参数校验失败","details":{"rating":"评分最高5分"}}

# 5. 查询某预约的评价
curl -s -X GET "localhost:8080/api/reviews/booking/1" \
  -H "Authorization: Bearer $TOKEN"
# → 200 {"code":"SUCCESS","data":{"rating":4,"comment":"设备运行良好",...}}

# 6. 评价列表
curl -s -X GET "localhost:8080/api/reviews?pageNum=1&pageSize=10" \
  -H "Authorization: Bearer $TOKEN"
# → 200 {"code":"SUCCESS","data":{"records":[...],"total":N}}

# 7. 删除评价
curl -s -X DELETE localhost:8080/api/reviews/1 \
  -H "Authorization: Bearer $TOKEN"
# → 200 {"code":"SUCCESS","message":"删除成功"}

# 8. 前端验证
# 浏览器访问 http://localhost:5173/reviews/mine
# 预期：表格展示评价记录，el-rate 显示评分，可点击删除按钮
```

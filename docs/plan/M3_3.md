## 二、实现计划

### 执行顺序

```
API 模块 (courses.ts + notices.ts)
  → NoticesView (M3-11/12 — 最独立)
    → AdminCoursesView (M3-8 — 课程 CRUD)
      → MyScheduleView (M3-9 — 依赖 courses API)
        → Router + Menu (统一收尾)
```

### 关键设计决策

| 决策                                                    | 理由                                                                              |
| ------------------------------------------------------- | --------------------------------------------------------------------------------- |
| 课程管理用 `el-input` + `isValidSemester()` 校验学期    | 学期格式固定为 `YYYY-YYYY-[12]`，无需自定义 semester picker；共享工具函数可复用   |
| 时间选择器用 `el-time-select` (08:00-22:00, 30min step) | 与预约时段精度一致，30 分钟为最小粒度                                             |
| 课程管理名称和教师筛选在前端做客户端过滤                | 后端 `/courses` API 不支持 name/teacherName 直接筛选，客户端过滤避免额外 API 改动 |
| 课表用 CSS Grid 而非 el-table                           | 高校课表标准布局是周历网格，课程卡片需按时间和星期二维定位，表格无法实现          |
| 课程卡片背景色用固定 8 色池 + 课程名哈希取模            | 颜色数量可控，不引入额外依赖                                                      |
| STUDENT 角色课表需输入班级                              | User 表无 className 字段，与后端 `/mine` API 设计一致                             |
| 通知用卡片而非表格                                      | 通知是内容驱动数据，卡片比表格更适合阅读；优先级色条提供视觉层次                  |
| M3-12 不单独建组件                                      | NoticesView 内三个 dialog 覆盖详情需求，单独建组件仅增加文件数                    |
| 通知 `NoticeUpdateRequest` 新增 `labId`                 | 编辑通知时需支持修改关联实验室，与后端 DTO 对齐                                   |

### M3-8 (AdminCoursesView) 验证

1. **列表展示** — 进入页面，课程表格正确加载（含课程名、实验室、教师、学期、上课时间、班级）
2. **实验室筛选** — 选择实验室 → 查询 → 仅显示该实验室课程
3. **名称筛选** — 输入课程名关键词 → 回车 → 列表过滤
4. **教师筛选** — 输入教师名 → 回车 → 列表过滤
5. **重置** — 点击重置 → 全部筛选清空 → 显示全部课程
6. **新增课程** — 点击"新增课程" → 弹窗 → 填写必填字段 → 提交 → toast "创建成功" → 列表刷新
7. **学期校验** — 学期输入非法格式（如 "abc"） → 失去焦点 → 行内错误提示"学期格式无效，应为 YYYY-YYYY-[12]"
8. **时间约束** — 开始时间 >= 结束时间 → 提交 → toast "开始时间必须早于结束时间"
9. **日期约束** — 开始日期 > 结束日期 → 提交 → toast "开始日期不能晚于结束日期"
10. **编辑课程** — 点击编辑 → 弹窗预填数据 → 修改后保存 → toast "更新成功"
11. **删除课程** — 点击删除 → 确认弹窗 → 确认 → toast "删除成功"
12. **上课时间显示** — 表格中上课时间列显示格式为"周三 08:00-10:00"
13. **分页** — 课程超过 20 条时分页器正确显示

### M3-9 (MyScheduleView) 验证

1. **STUDENT 角色** — 用学生账号登录 → 进入 `/schedule` → 显示班级输入框 → 输入班级 → 查询 → 课表网格渲染
2. **TEACHER 角色** — 用教师账号登录 → 进入 `/schedule` → 不显示班级输入框 → 查询 → 显示所教课程
3. **ADMIN 角色** — 管理员登录 → 查询 → 显示全部课程
4. **学期筛选** — 输入学期 → 查询 → 仅显示该学期课程
5. **网格布局** — 7 列（周一~~周日表头）+ 时间标签（08:00~~22:00 整点显示）
6. **课程卡片** — 卡片按 dayOfWeek 和 startTime/endTime 正确定位，背景色区分不同课程
7. **卡片内容** — 每张卡片显示课程名、班级、教师、时间段
8. **空状态** — 无课程或 STUDENT 未输入班级 → 显示 el-empty "暂无课程"

### M3-11/12 (NoticesView) 验证

1. **卡片列表** — 进入页面，通知卡片正确加载，左侧优先级色条颜色正确（URGENT=红 HIGH=橙 NORMAL=蓝 LOW=灰）
2. **类型筛选** — 选择类型 → 查询 → 仅显示对应类型通知
3. **优先级筛选** — 选择优先级 → 查询 → 仅显示对应优先级通知
4. **重置** — 点击重置 → 全部筛选清空 → 显示全部通知
5. **发布通知（TEACHER/ADMIN）** — 点击"发布通知" → 弹窗 → 填写标题+内容 → 提交 → toast "发布成功"
6. **编辑通知（TEACHER/ADMIN）** — 卡片点击编辑 → 弹窗预填 → 修改 → toast "更新成功"

8. **详情弹窗** — 点击卡片 → 详情弹窗显示标题、标签、发布者、时间、关联实验室（可点击跳转）、正文全文
9. **关联实验室跳转** — 详情弹窗中点击实验室链接 → 跳转到 `/labs/:id`
10. **STUDENT 无管理按钮** — 学生查看通知时卡片不显示编辑/删除按钮，工具栏无"发布通知"按钮

12. **分页** — 通知超过 20 条时分页器正确显示

**`api/client.ts`** — Axios 实例：

- `baseURL: '/api'`，timeout 10s
- 请求拦截器：从 `localStorage.getItem('token')` 读取，设置 `Authorization: Bearer <token>`
- 响应拦截器：`code !== 'SUCCESS'` → `Promise.reject(new Error(message))` 并弹出 `ElMessage.error`
- 401 时清除 `localStorage` 中的 `token` 和 `userInfo`，`window.location.href = '/login'` 跳转

**`api/auth.ts`** — 4 个认证 API 封装：

| 函数                   | 方法 | 路径                    | 返回类型                 |
| ---------------------- | ---- | ----------------------- | ------------------------ |
| `register(data)`       | POST | `/auth/register`        | `Promise<void>`          |
| `login(data)`          | POST | `/auth/login`           | `Promise<LoginResponse>` |
| `getMe()`              | GET  | `/auth/me`              | `Promise<UserInfo>`      |
| `changePassword(data)` | PUT  | `/auth/change-password` | `Promise<void>`          |

**`stores/auth.ts`** — Pinia setup store：

```
state:
  token       ← localStorage.getItem('token')
  userInfo    ← JSON.parse(localStorage.getItem('userInfo'))

actions:
  login(username, password)   → authApi.login() → 存 token + userInfo 到 state + localStorage
  logout()                    → 清 token + userInfo + localStorage
  fetchUser()                 → authApi.getMe() → 刷新 userInfo

getters:
  isLoggedIn    → !!token
  isAdmin       → userInfo.role === 'ADMIN'
  isTeacher     → userInfo.role === 'TEACHER'
  userRole      → userInfo.role
```

beforeEach 守卫执行流程：

```
1. 有 token 无 userInfo → fetchUser() 恢复，失败则 logout() 并跳转 /login
2. matched 路由链中任意 meta.requiresAuth && !isLoggedIn → /login?redirect=原路径
3. meta.roles 不包含当前角色 → /
4. 已登录访问 /login → /
```

### 关键设计决策

| 决策                                                  | 理由                                                                                                                |
| ----------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------- |
| token 存 localStorage 而非 cookie                     | 前端 SPA 通过 Axios 拦截器手动附加 Header，避免 CSRF 问题                                                           |
| `auth` store 使用 Setup Store（组合式 API）           | 与 Vue 3 `<script setup>` 风格一致                                                                                  |
| 响应拦截器直接 `ElMessage.error()`                    | 全局统一错误提示，避免每个调用方重复处理                                                                            |
| 注册接口直接调 `authApi.register()` 而非 store action | register 不修改 store 状态，无需经过 store                                                                          |
| store 从 localStorage 恢复 userInfo                   | 刷新页面后无需重新调 `/auth/me`，直接从缓存恢复                                                                     |
| 类型注解使用 `AxiosResponse<ApiResponse<T>>`          | 消除隐式 `any`，通过 `vue-tsc` 类型检查                                                                             |
| 不使用 `router.push()` 做 401 跳转                    | `client.ts` → `router` → `stores/auth` → `api/auth` → `client.ts` 形成循环依赖；`window.location.href` 硬刷新更彻底 |
| `beforeEach` 中先恢复 userInfo 再检查权限             | 页面刷新时 token 在 localStorage 但 userInfo 为 null，需先调 `/auth/me` 恢复                                        |
| fetchUser 失败时调用 logout()                         | token 可能已过期，logout() 清除所有残留状态                                                                         |
| 创建/编辑共用同一个 Dialog                            | 减少模板重复，通过 `isEdit` 计算属性区分校验规则和表单字段                                                          |
| 编辑时不显示密码字段                                  | 后端 `UserUpdateRequest` 不含 password，密码修改走独立的 `/change-password` 接口                                    |
| `USER_ROLE_LABELS` 生成下拉选项                       | 复用 shared 包中的标签映射，避免硬编码                                                                              |
| 登录/注册在嵌套路由外部                               | AppLayout 仅包裹需认证的页面，登录/注册保持独立                                                                     |
| `el-menu` 用 `v-if` 过滤菜单项                        | 让 `visibleMenuItems` 计算属性按角色过滤，后续 M2/M3 扩展菜单项只需追加数组元素                                     |
| `to.matched.some()` 检查 requiresAuth                 | 嵌套路由中 meta 可能在父路由或子路由上，需要遍历整条匹配链                                                          |
| `AuthenticationEntryPoint` 返回 JSON                  | 后端 API 不是表单登录，401 应返回结构化 JSON 而非 HTML 重定向                                                       |


### 认证 + 权限 curl 验证

```bash
# 获取 ADMIN token
ADMIN_LOGIN=$(curl -s -X POST localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"devadmin","password":"admin123"}')
TOKEN=$(echo "$ADMIN_LOGIN" | sed 's/.*"token":"\([^"]*\)".*/\1/')

# 用户列表
curl -s -X GET "localhost:8080/api/users?pageNum=1&pageSize=10" \
  -H "Authorization: Bearer $TOKEN"
# → 200 {"code":"SUCCESS","data":{"records":[...],"total":N,...}}

# 非 ADMIN → 403
curl -s -X GET "localhost:8080/api/users" \
  -H "Authorization: Bearer <STUDENT_TOKEN>"
# → 403 {"code":"FORBIDDEN","message":"无权限","details":null}
```

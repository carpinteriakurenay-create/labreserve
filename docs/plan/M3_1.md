## 二、实现计划

### 执行顺序

```
M3-1 (Equipment)
  DTO → EquipmentService → EquipmentController → equipment.ts API → AdminEquipmentView
                                                                          ↓
M3-2 (Borrow)                                                     路由 + 菜单
  DTO → BorrowService → BorrowController → borrows.ts API → 3 个 View 页面
```

M3-1 和 M3-2 的 DTO/Service/Controller 可以独立开发，前端 View 依赖各自的 API 模块，路由和菜单在最后统一修改。

### 状态机（Borrow）

```
PENDING ──通过──▶ BORROWING ──归还──▶ RETURNED
   │
   └──驳回──▶ REJECTED
```

设计决策：

- **审批通过直接进入 BORROWING**（跳过 APPROVED 中间状态），减少一步操作
- **归还时设备状态自动恢复为 AVAILABLE**
- **借用申请时校验设备 status=AVAILABLE**，不可用直接拒绝

### 关键设计决策

| 决策                                                     | 理由                                                                     |
| -------------------------------------------------------- | ------------------------------------------------------------------------ |
| Entity 和 Mapper 已在 M1 阶段建好                        | M1-3 批量生成了全部 13 个 Entity + Mapper                                |
| 分页用 MyBatis-Plus `IPage` 而非 cursor-based            | 与现有 LabService / BookingService 保持一致，前端用 `pageNum`/`pageSize` |
| VO 中填充关联名称（labName, equipmentName, userName）    | 仿照 `BookingService.populateJoinedFields()` 模式，batch select 后填充   |
| 创建时 serialNumber 唯一性校验                           | `selectCount` + `SERIAL_NUMBER_EXISTS` 异常                              |
| 更新用 `LambdaUpdateWrapper` 仅 set 非 null 字段         | 支持部分更新，与 PUT 语义一致                                            |
| 借用审批复用 `ApprovalRequest`                           | 与 Booking 审批接口保持统一参数格式                                      |
| 归还接口用 `@RequestBody Map<String, String>`            | actualReturn 可选，避免为单个字段创建 DTO                                |
| `EQUIPMENT_UNAVAILABLE` 已在 GlobalExceptionHandler      | M1-5 阶段预置，映射到 409 CONFLICT                                       |
| 前端 API 模块一个资源一个文件                            | 与 `labs.ts` / `users.ts` / `bookings.ts` 模式一致                       |
| 前端 View 均用 `<script setup lang="ts">` + Element Plus | 与 `AdminUsersView.vue` 模式一致                                         |

## 三、验证方案

### M3-1 验证

> **前置条件：** 数据库已初始化，有可用的 ADMIN token。

```bash
ADMIN_TOKEN="<admin token>"

# 1. 登记设备（ADMIN）
curl -s -X POST localhost:8080/api/equipment \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -d '{"labId":"1","name":"FPGA 开发板","model":"Xilinx Artix-7","serialNumber":"FPGA-2026-001","description":"第3批次采购"}'
# → 201 {"code":"SUCCESS","message":"创建成功","data":{"id":1,"name":"FPGA 开发板","status":"AVAILABLE",...}}

# 2. 重复序列号 → 409
curl -s -X POST localhost:8080/api/equipment \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -d '{"labId":"1","name":"FPGA 开发板 v2","model":"Xilinx","serialNumber":"FPGA-2026-001","description":""}'
# → 409 {"code":"SERIAL_NUMBER_EXISTS","message":"设备序列号已存在","details":null}

# 3. 列表（分页 + 按名称搜索）
curl -s -X GET "localhost:8080/api/equipment?name=FPGA&pageNum=1&pageSize=10" \
  -H "Authorization: Bearer $ADMIN_TOKEN"
# → 200 {"code":"SUCCESS","data":{"records":[...],"total":1,...}}

# 4. 按实验室筛选
curl -s -X GET "localhost:8080/api/equipment?labId=1" \
  -H "Authorization: Bearer $ADMIN_TOKEN"
# → 200（仅 labId=1 的设备）

# 5. 按状态筛选
curl -s -X GET "localhost:8080/api/equipment?status=AVAILABLE" \
  -H "Authorization: Bearer $ADMIN_TOKEN"
# → 200（仅 AVAILABLE 设备）

# 6. 设备详情
curl -s -X GET localhost:8080/api/equipment/1 \
  -H "Authorization: Bearer $ADMIN_TOKEN"
# → 200 {"code":"SUCCESS","data":{"id":1,"labName":"计算机组成原理实验室",...}}

# 7. 更新设备
curl -s -X PUT localhost:8080/api/equipment/1 \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -d '{"name":"FPGA 开发板（升级）","status":"MAINTENANCE"}'
# → 200 {"code":"SUCCESS","message":"更新成功","data":{"name":"FPGA 开发板（升级）","status":"MAINTENANCE",...}}

# 8. 更新状态
curl -s -X PUT "localhost:8080/api/equipment/1/status?status=AVAILABLE" \
  -H "Authorization: Bearer $ADMIN_TOKEN"
# → 200 {"code":"SUCCESS","data":null}

# 9. 软删除
curl -s -X DELETE localhost:8080/api/equipment/1 \
  -H "Authorization: Bearer $ADMIN_TOKEN"
# → 200 {"code":"SUCCESS","message":"删除成功","data":null}
# 再次 GET /api/equipment/1 → 404

# 10. 非 ADMIN 访问 → 403
curl -s -X POST localhost:8080/api/equipment \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $STUDENT_TOKEN" \
  -d '{"labId":"1","name":"test","serialNumber":"SN-001"}'
# → 403
```

### M3-2 验证

```bash
STUDENT_TOKEN="<student token>"
ADMIN_TOKEN="<admin token>"

# 1. 借用申请
curl -s -X POST localhost:8080/api/borrows \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $STUDENT_TOKEN" \
  -d '{"equipmentId":"1","borrowDate":"2026-07-01","expectedReturn":"2026-07-05","purpose":"课程设计需要"}'
# → 201 {"code":"SUCCESS","message":"借用申请提交成功","data":{"id":1,"status":"PENDING",...}}

# 2. 设备不可用（已被借出或维修中） → 409
curl -s -X POST localhost:8080/api/borrows \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $STUDENT_TOKEN" \
  -d '{"equipmentId":"2","borrowDate":"2026-07-01","expectedReturn":"2026-07-05","purpose":"test"}'
# → 409 {"code":"EQUIPMENT_UNAVAILABLE","message":"设备不可用，已被借出或维修中"}

# 3. 我的借用列表
curl -s -X GET "localhost:8080/api/borrows/mine" \
  -H "Authorization: Bearer $STUDENT_TOKEN"
# → 200 {"code":"SUCCESS","data":{"records":[...],"total":1,...}}

# 4. 审批通过（ADMIN/TEACHER）
curl -s -X PUT localhost:8080/api/borrows/1/approve \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -d '{"approved":true}'
# → 200 {"code":"SUCCESS","message":"审批通过","data":{"status":"BORROWING",...}}
# 验证设备状态已变更为 BORROWED：
curl -s -X GET localhost:8080/api/equipment/1 \
  -H "Authorization: Bearer $ADMIN_TOKEN"
# → status: "BORROWED"

# 5. 审批驳回
curl -s -X PUT localhost:8080/api/borrows/2/approve \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -d '{"approved":false,"rejectReason":"设备已损坏"}'
# → 200 {"code":"SUCCESS","message":"已拒绝","data":{"status":"REJECTED","rejectReason":"设备已损坏"}}

# 6. 已处理 → 不可重复审批
curl -s -X PUT localhost:8080/api/borrows/1/approve \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -d '{"approved":true}'
# → 409 {"code":"ALREADY_PROCESSED","message":"该借用申请已被处理"}

# 7. 确认归还
curl -s -X PUT localhost:8080/api/borrows/1/return \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -d '{"actualReturn":"2026-07-04"}'
# → 200 {"code":"SUCCESS","message":"归还成功","data":null}
# 验证设备状态已恢复：
curl -s -X GET localhost:8080/api/equipment/1 \
  -H "Authorization: Bearer $ADMIN_TOKEN"
# → status: "AVAILABLE"

# 8. 状态校验：非 BORROWING 不可归还
curl -s -X PUT localhost:8080/api/borrows/1/return \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $ADMIN_TOKEN"
# → 400 {"code":"INVALID_STATUS","message":"只有借用中的记录可以归还"}

# 9. 借用详情（含关联名称）
curl -s -X GET localhost:8080/api/borrows/1 \
  -H "Authorization: Bearer $ADMIN_TOKEN"
# → 200 {"code":"SUCCESS","data":{"equipmentName":"FPGA 开发板","userName":"张三",...}}

# 10. 全部借用列表
curl -s -X GET "localhost:8080/api/borrows?status=BORROWING" \
  -H "Authorization: Bearer $ADMIN_TOKEN"
# → 200（仅 BORROWING 状态记录）
```

| 不暴露 `/mine`                                    | 通知是全局可见的，API 设计未定义该端点 |

| 新建   | `apps/api/src/main/java/com/labreserve/controller/CourseController.java` |

6 个端点：

| 方法   | 路径                      | 说明                                                       | 认证          |
| ------ | ------------------------- | ---------------------------------------------------------- | ------------- |
| GET    | `/api/courses`            | 列表（semester, teacherId, labId, className 筛选）         | 需 JWT        |
| POST   | `/api/courses`            | 创建课程（含 semester/dayOfWeek/time/date 校验）→ 201      | TEACHER/ADMIN |
| GET    | `/api/courses/mine`       | 我的课表（学生按 className，教师按 teacherId，管理员全量） | 需 JWT        |
| GET    | `/api/courses/{courseId}` | 课程详情（含 labName, teacherName）                        | 需 JWT        |
| PUT    | `/api/courses/{courseId}` | 更新课程（部分字段，semester 不允许修改）                  | TEACHER/ADMIN |
| DELETE | `/api/courses/{courseId}` | 删除课程（软删除）                                         | TEACHER/ADMIN |

关键设计决策：

| 决策                                          | 理由                                                                     |
| --------------------------------------------- | ------------------------------------------------------------------------ |
| 分页用 MyBatis-Plus `IPage` 而非 cursor-based | 与现有所有 Service 保持一致（Equipment/Borrow/Booking 均用 offset 分页） |
| `/mine` 学生需传 `className` 参数             | User 表无 className 字段，学生通过班级名匹配课程                         |
| `/mine` 教师按 `teacherId = userId`           | 教师看到自己任教的全部课程                                               |
| `/mine` 管理员全量返回                        | 管理员等同 listCourses                                                   |
| semester 格式校验 `\d{4}-\d{4}-[12]`          | 与 `packages/shared/src/utils.ts` 的 `isValidSemester()` 一致            |
| dayOfWeek 校验 1-7                            | 防止非法星期值                                                           |
| startTime < endTime, startDate <= endDate     | 保证课程时间范围的合法性                                                 |
| UpdateRequest 不含 semester                   | 课程学期创建后不可变更（业务规则）                                       |
| 更新时需重新验证时间范围                      | 防止部分更新后的时间范围不合法                                           |

## 二、实现计划

### 执行顺序

```
Notice DTOs (3 files) → NoticeService → NoticeController
                                              ↓
Course DTOs (3 files) → CourseService → CourseController
                                              ↓
                              GlobalExceptionHandler (追加错误码)
```

Notice 和 Course 两个模块相互独立，可并行实现。GlobalExceptionHandler 在最后修改，一次性追加所有新错误码。

### 关键设计决策

| 决策                                                   | 理由                                                                                 |
| ------------------------------------------------------ | ------------------------------------------------------------------------------------ |
| 仅实现后端 API，不含前端页面                           | 本任务范围是 M3-7 + M3-10，前端在 M3-8/M3-9/M3-11/M3-12                              |
| `@PreAuthorize` 粒度在方法级                           | 与现有 Controller（EquipmentController, BorrowController）一致                       |
| Service 不 extend IService/ServiceImpl                 | 与现有 Service 模式一致，手动编写 LambdaQueryWrapper 逻辑                            |
| `toVO()` + `populateJoinedFields()` 模式               | 与 EquipmentService/BorrowService 保持一致的转换 + 关联填充模式                      |
| Notice 的 type/priority 用 `NoticeType.valueOf()` 校验 | 无效枚举值 → `IllegalArgumentException` → `BusinessException("INVALID_STATUS", ...)` |
| Course 不修改 BookingService                           | BookingService 已直接调用 `courseMapper.selectList()` 做冲突检测，无需改动           |

### 获取测试 Token

```bash
# 注册管理员
curl -s -X POST localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"devadmin","password":"admin123","realName":"Dev Admin"}'

# 提权为 ADMIN（通过 DB）
docker exec -it mysql labreserve_dev \
  -e "UPDATE users SET role='ADMIN' WHERE username='devadmin';"

# 获取 ADMIN token
ADMIN_LOGIN=$(curl -s -X POST localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"devadmin","password":"admin123"}')
TOKEN=$(echo "$ADMIN_LOGIN" | sed 's/.*"token":"\([^"]*\)".*/\1/')
echo "Token: ${TOKEN:0:20}..."

# 注册教师
curl -s -X POST localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"teacher1","password":"pass123","realName":"李教授"}'

docker exec -it mysql labreserve_dev \
  -e "UPDATE users SET role='TEACHER' WHERE username='teacher1';"

  -H "Content-Type: application/json" \
```

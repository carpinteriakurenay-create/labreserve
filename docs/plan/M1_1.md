关键设计决策：

| 决策                                                              | 理由                                                                            |
| ----------------------------------------------------------------- | ------------------------------------------------------------------------------- |
| 分页用 MyBatis-Plus `IPage` 而非 cursor-based                     | M1 阶段简化，前端用 `pageNum`/`pageSize` 即可                                   |
| `PaginationInnerInterceptor` 需单独依赖 `mybatis-plus-jsqlparser` | 3.5.9+ 拆分出的模块                                                             |
| 更新用 `LambdaUpdateWrapper` 仅 set 非 null 字段                  | 支持部分更新，与 PUT 语义一致                                                   |
| 创建时 username 唯一性检查                                        | `selectCount` + `USERNAME_EXISTS` 异常                                          |
| `toUserInfo()` 私有方法映射 Entity → DTO                          | 与 AuthService 中映射逻辑一致，role 用 `.name()` 转 String，enabled int→Boolean |
| `PasswordEncoder` 构造器注入                                      | 复用 SecurityConfig 中的 BCryptPasswordEncoder Bean                             |

## 二、实现计划

### 执行顺序

```
M1-1 (init.sql) ──────────────┐
                               ├──> M1-5 (AuthService) --> M1-6 (AuthController) --> M1-7 (UserService+Controller)
M1-2 (pom.xml + yml) ──> M1-3 ─┤
                    └─> M1-4 ──┘
```

M1-1 和 M1-2 并行开始，M1-3 依赖 M1-1，M1-4 依赖 M1-2，M1-5 依赖 M1-3 + M1-4，M1-6 依赖 M1-5，M1-7 依赖 M1-5。

### 关键设计决策

| 决策                                     | 理由                                                                               |
| ---------------------------------------- | ---------------------------------------------------------------------------------- |
| 软删除 `deleted TINYINT DEFAULT 0`       | 与 `application.yml` 的 `logic-delete-value: 1` / `logic-not-delete-value: 0` 一致 |
| ENUM 列 → `VARCHAR`                      | MySQL ENUM 扩展需 ALTER TABLE，VARCHAR 更灵活                                      |
| 不建外键约束                             | MyBatis-Plus 不管理关系，关联查询在 Service 层手动 JOIN                            |
| JJWT 0.12.6 + HS256                      | Spring Boot 3.4 兼容版本                                                           |
| `@TableLogic(value = "0", delval = "1")` | 与 SQL DDL 的 `deleted TINYINT DEFAULT 0` 对齐                                     |
| DTO 中 `id` 用 `Long`                    | Jackson 序列化为 JSON number，比 String 更符合 Java 习惯                           |
| `UserId` 存入 JWT `sub` claim            | `/auth/me` 直接用 `getPrincipal()` 获取 userId，无需查库                           |
| 注册默认角色 `STUDENT`                   | 符合 F-01 需求                                                                     |
| `ApiResponse<T>` 统一成功包              | 与 `ApiError` 对仗，`code="SUCCESS"`，data 泛型                                    |
| Controller 零 try-catch                  | `GlobalExceptionHandler` 已拦截所有异常                                            |

### M1-5 + M1-6 联合验证

> **前置条件：** `init.sql` 中的 BCrypt 哈希是预生成字面量（非真实哈希），无法直接用于登录。需先通过 API 注册生成正确哈希的账号。

```bash
# 注册
curl -s -X POST localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"test001","password":"pass123","realName":"测试"}'
# → 201 {"code":"SUCCESS","message":"注册成功","data":null}

# 重复注册 → 409
curl -X POST localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"test001","password":"pass123","realName":"测试"}'
# → 409 {"code":"USERNAME_EXISTS","message":"用户名已被注册","details":null}

# 登录
curl -X POST localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"test001","password":"pass123"}'
# → 200 {"code":"SUCCESS","message":"登录成功","data":{"token":"eyJ...","expiresIn":86400,"user":{...}}}

# 错误密码 → 401
curl -X POST localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"test001","password":"wrong"}'
# → 401 {"code":"INVALID_CREDENTIALS","message":"用户名或密码错误","details":null}

# 获取当前用户（用上一步的 token）
curl -X GET localhost:8080/api/auth/me \
  -H "Authorization: Bearer <token>"
# → 200 {"code":"SUCCESS","message":"操作成功","data":{...}}

# 无 token → 401
curl -X GET localhost:8080/api/auth/me
# → 403 (被 SecurityConfig 拒绝)
```

### M1-7 验证

> **前置条件：** 本地 MySQL 默认无 ADMIN 用户。先注册一个用户，再通过 DB 提升为 ADMIN：
>
> ```bash
> # 注册+提权
> curl -s -X POST localhost:8080/api/auth/register -H "Content-Type: application/json" \
>   -d '{"username":"devadmin","password":"admin123","realName":"Dev Admin"}'
> mysql -ulabreserve -plabreserve123 labreserve_dev \
>   -e "UPDATE users SET role='ADMIN' WHERE username='devadmin';"
> ```
>
> 之后用 `devadmin` 的 token 测试全部接口。

```bash
# 获取 ADMIN token
ADMIN_LOGIN=$(curl -s -X POST localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"devadmin","password":"admin123"}')
# 提取 token（兼容无 jq 环境，用 sed）
TOKEN=$(echo "$ADMIN_LOGIN" | sed 's/.*"token":"\([^"]*\)".*/\1/')
echo "Token: ${TOKEN:0:20}..."

# 1. 列表（分页）
curl -s -X GET "localhost:8080/api/users?pageNum=1&pageSize=10" \
  -H "Authorization: Bearer $TOKEN"
# → 200 {"code":"SUCCESS","data":{"records":[...],"total":N,"current":1,"pages":N}}

# 2. 按角色筛选
curl -s -X GET "localhost:8080/api/users?role=STUDENT" \
  -H "Authorization: Bearer $TOKEN" | grep -o '"total":[0-9]*'
# → "total":N（仅 STUDENT 角色）

# 3. 创建用户 → 201
curl -s -X POST localhost:8080/api/users \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"username":"newuser","password":"pass123456","realName":"New User","role":"TEACHER"}'
# → 201 {"code":"SUCCESS","message":"创建成功","data":{"id":N,"username":"newuser","role":"TEACHER",...}}

# 4. 重复创建 → 409
curl -s -X POST localhost:8080/api/users \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"username":"newuser","password":"pass123456","realName":"Dup"}'
# → 409 {"code":"USERNAME_EXISTS","message":"用户名已被使用","details":null}

# 5. 详情
curl -s -X GET localhost:8080/api/users/<new userId> \
  -H "Authorization: Bearer $TOKEN"
# → 200 {"code":"SUCCESS","data":{"id":N,"username":"newuser",...}}

# 6. 更新（部分字段）
curl -s -X PUT localhost:8080/api/users/<new userId> \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"realName":"Updated User","email":"updated@test.com"}'
# → 200 {"code":"SUCCESS","message":"更新成功","data":{"realName":"Updated User","email":"updated@test.com",...}}

# 7. 启用/禁用切换
curl -s -X PUT localhost:8080/api/users/<userId>/toggle-enabled \
  -H "Authorization: Bearer $TOKEN"
# → 200 {"code":"SUCCESS","message":"操作成功","data":null}
# 再次 GET 用户详情 → enabled 字段已翻转

# 8. 软删除
curl -s -X DELETE localhost:8080/api/users/<userId> \
  -H "Authorization: Bearer $TOKEN"
# → 200 {"code":"SUCCESS","message":"删除成功","data":null}
# 再次 GET 详情 → 404 {"code":"NOT_FOUND","message":"用户不存在"}

# 9. 非 ADMIN 访问 → 403
STUDENT_LOGIN=$(curl -s -X POST localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"newuser","password":"pass123456"}')
STUDENT_TOKEN=$(echo "$STUDENT_LOGIN" | sed 's/.*"token":"\([^"]*\)".*/\1/')
curl -s -X GET "localhost:8080/api/users" \
  -H "Authorization: Bearer $STUDENT_TOKEN"
# → 403 {"code":"FORBIDDEN","message":"无权限","details":null}

# 10. 创建时用户名缺失 → 400
curl -s -X POST localhost:8080/api/users \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"password":"abc","realName":"NoName"}'
# → 400 {"code":"VALIDATION_ERROR","message":"请求参数校验失败","details":{"username":"用户名不能为空"}}
```
